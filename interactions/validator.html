--- layout: interactions title: Validator name: validator interaction_selected: selected description:

When a user types incorrect information into the input field, on clicking outside of the input, an error attribute will be assigned to the input. Also, optionally, an error message will appear explaining the error.

basic_tab: data-ur-set="validator": add this attribute to a div that wraps the whole interaction data-ur-validator-component="*": add this attribute to the input in the set; possible values are email, ccnum, notblank (note that there can be only one input per set) data-ur-validator-component="error": add this optional attribute after the input; it will data-ur-validator-min / data-ur-validator-max: can be used to specify a minimum or maximum number for the input data-ur-validator-ccard-type="*": attribute is added to the credit card input, giving the type of credit card entered advanced_tab: Required: email/ccnum/notblank: - add this attribute to the input node Optional: Error: - add this attribute to a div after the input, but within the set ---

Demonstration

In the example below, the first input cannot be blank. It does not have an error component so no message shows up. The other two examples require a valid e-mail and credit card number respectively. They do have the error component, so error messages appear.
