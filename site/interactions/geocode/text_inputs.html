Geolocation with Text Inputs

Description

This example illustrates that a select form is not necessary to fill a field. We've used a text input instead of a select for the state and country.

Demonstration

The demonstration activates on button click.

Example

The Code


<form data-ur-set="reverse-geocode" action="geocode_submit" method="get" accept-charset="utf-8" >
  <button  value="clickMe" type='button' data-ur-reverse-geocode-component="rg-button">Click here to find your location</button>
  <input type="text" name="addr_1" value="" id="street_address" placeholder="street address" data-ur-reverse-geocode-component="rg-street">
  <input type="text" name="city" value="" id="city" placeholder="city" data-ur-reverse-geocode-component="rg-city">
  <input type="text" name="zip" value="" id="zip" placeholder="zip" data-ur-reverse-geocode-component="rg-zip">
  <!-- State works with both inputs and select elements -->
  <input type="text" name="state" value="" id="state" placeholder="state" data-ur-reverse-geocode-component="rg-state">
  <!-- Country works with both inputs and select elements -->
  <input type="text" name="country" value="" id="country" placeholder="country" data-ur-reverse-geocode-component="rg-country">
</form>
